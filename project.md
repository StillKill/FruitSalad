# Fruit Salad (Фруктовый Салат) — Цифровая версия Point Salad на Phaser JS

## Описание
Настольная игра в жанре Set Collection, filler,  аналог Point Salad, только с фруктами вместо овощей, Web версия с локальным мультиплеером для 2-6 игроков (Hot Seat)

## Модули проекта

1 Игровое поле :
- Левая половину экрана - колоды и рынок, правая - зона игрока 
- Decks & Market
	- сетка под карты 3х3верхний ряд - места для трех колод карт, куда карты распределяются по ровну на старте игры
	- под каждой из колод по два слота для рынка карт
- зона игрока
	- ряд для салатов
	- 6 стопок под фрукты
	- счетчик для каждого фрукта

2 Карты
- размер карт PNG, 432 x 600, можно скейлить
- по 18 карт каждого вида фрукта. КИВИ / АПЕЛЬСИН / ЯБЛОКО / БАНАН / ЛАЙМ / МАНГО всего 6 видов
- 108 карт "салатов", это scoring карты

3. UI
- Диалоговое окно настроек при старте игры (settings). Указание количества и имен игроков
- панель управления ходом Controls. 
	- Кнопка подтвердить ход Confirm, активируется в стейте end_turn
	- Кнопка Reset, всегда активна, при нажатии сбрасывает ход игрока в начало, возвращает карты на место
-  строка с подсказкой для игрока с изменяющимся содержимым (подсказка, что нужно взять карты/карту, подтвердить или отменить ход). Находится под кнопками управления ходом
- панель с вкладками (с именами) игроков, активный игрок выделен цветом, но можно переключаться между игроками.
- сразу ниже  - панель с очками для каждого игрока
- Вкладка игрока содержит
	- панель иконок фруктов
	- под каждой иконкой поле с отображением текущего количества
	- рука игрока с картами салатов, которые забрал игрок, может скроллится вниз при большом количестве
- дебажное поле с логом в правой части экрана, текущий стейт и прочая полезная информация
- Возможно снизу дополнительный экран где дублируются на постоянной основе вкладки всех игроков, чтобы можно было их изучать одновременно (кол-во фруктов и их салаты)
- popup  с результатами игры

3.    Игровые стейты
- settings: настройка количества игроков и имен (2-6 игроков)
- setup: создание основного игрового поля и зон игроков, отбор карт в колоду в зависимости от количества игроков (количество карт берется из файла настроек), создание 3 коллод карт и рынка на поле
- turn: ждем инпута от игрока, игрок может кликнуть по карте на рынке или в колоде, если взял одну карту на рынке, то возможность брать из колоды блокируется.
- end_turn: переход в стейт из turn, когда игрок взял две карты с рынка или одну с колоды, можно жать confirm хода для передачи хода следующему игроку.
- refresh: 
	- рынок автоматически пополняется после хода игрока, если есть пустые места из соответствующей коллоды
	- если одна из коллод кончилась наибольшая коллода делится пополам и нижняя половина встаёт на пустое место
	- если все коллоды иссякли - - переход в end_game
- end_game подсчет очков для каждого игрока. поп ап с объявлением победителя.

3. Scoring
- карты салатов, они же scoring карты по принципу подсчет очков бывают нескольких типов.
	- карты, требующие сравнения между игроками в end_game дают очки один раз за выполнение условия
		- majority - больше всех фруктов одного вида - 10 очк
		- minority - меньше всех фруктов одного вида - 7 очк
		- poor - меньше всего карт фруктов - 7 очк
		- wealth - больше всего карт фруктов - 10 очк
	- карты с личными условиями
		- Четное или неченое количество фруктов указаного вида - Четное 7 Нечетное 3
		- Два Или более овоща одного вида - 3 очка за вид
		- три Или более овоща одного вида - 5 очка за вид
		- отсутствующие виды овощей - 5 за вид
	- карты наборов - сетов
		- два фрукта одного вида - 5
		- три фрукта одного вида - 8
		- Пара разных фруктов - 5
		- Три фрукта разных видов - 8
		- Набор из 6 овощей - 12
	 - карты которые дают указанное количество очков за каждый указанный фрукт, могут отнимать очки
		 - 2 очка за фрукт одного вида
		 - 1 первому, 1 второму
		 - +3 первому, -2 второму
		 - +3 первому, -1 второму, - 1 третьему
		 - +2 первому, +1 второму, - 2 третьему
		 - +4 первому, -2 второму, - 2 третьему
		 - +2 первому, +2 второму, - 4 третьему
